# Contributing

Thank you for your interest in contributing to Awesome Autonomous Operation!

## Guidelines

### Adding a Resource

- Make sure the resource is relevant to autonomous operations, AIOps, observability, telecom automation, or related fields.
- Only suggest resources that are actively maintained and of high quality.
- Do not add deprecated, unmaintained, or abandoned projects.
- Use the following format:

```
- [Name](URL) - Short, objective description.
```

- Keep descriptions concise (one sentence) and avoid marketing language.
- Add new entries at the end of the relevant section.
- Check that the URL is valid and accessible.

### Adding an Analysis Page

Analysis pages live in `analysis/en/` and `analysis/ja/`. Each should include:

- Overview of the resource
- Main categories and key projects
- Relevance & Advantages for autonomously-io

### Pull Request Process

1. Fork the repository.
2. Create a new branch (`git checkout -b add-resource-name`).
3. Make your changes.
4. Run `npx awesome-lint` to check for issues.
5. Submit a pull request with a clear description of what you added and why.

### Quality Standards

- **No duplicates** — Check if the resource is already listed.
- **No self-promotion** — Do not add your own project unless it is widely recognized.
- **Objectivity** — Descriptions should be factual, not promotional.
- **English and Japanese** — If adding an analysis page, provide both `en/` and `ja/` versions when possible.

### Updating Existing Entries

- Fix broken links.
- Update descriptions if a project has significantly changed scope.
- Remove entries for projects that are no longer maintained.

## Code of Conduct

Be respectful, constructive, and inclusive. We welcome contributions from everyone.
